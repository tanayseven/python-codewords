The Zen of Python, by Tim Peters

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
Special cases aren't special enough to break the rules.
Although practicality beats purity.
Errors should never pass silently.
Unless explicitly silenced.
In the face of ambiguity, refuse the temptation to guess.
There should be one-- and preferably only one --obvious way to do it.
Although that way may not be obvious at first unless you're Dutch.
Now is better than never.
Although never is often better than *right* now.
If the implementation is hard to explain, it's a bad idea.
If the implementation is easy to explain, it may be a good idea.
Namespaces are one honking great idea -- let's do more of those!

The User Stories for the application

As a <Role> I want to <Action> in order to <Purpose>
As a User I want to add items to a cart entry in the cart in order to to know how many items of that article we are buying
As a User I want to see the number of cart entries in the cart in order to know how many total items I'm buying
As a User I want to see list of all the cart entries in the cart in order to add or remove or make changes to the items in the cart
As a User I want to remove a cart entry from the cart in order to cancel the buying of that item
As a User I want to remove an item from the cart entry in order to reduce the number of that items I'm going to buy
As a User I want to see total number of items in a single cart entry in order to know how many items of the same product I'm buying
As a User I want to see the total price of all items at every cart entry in the cart in order to verify that I want to pay that much for the set of items
As a User I want to see the item's individual cost in the cart entry in the cart in order to verify that I want to pay that much for single item
As a User I want to see the deducted discounted (in %ge) cost of item in the cart with the original cost in order to see how the amount of difference in the actual cost and the discounted one for a single item
As a User I want to see the deducted discounted (in %ge) cost of cart entry in the cart with the original cost in order to see how the amount of difference in the actual cost and the discounted one for the set of items
As a User I want to see the total cost of all the cart entries in the cart in order to verify that I have all the necessary items added to be purchased and have the amount to pay
